# DSM-AE MVP Implementation Plan

> **For agentic workers:** Implement task-by-task. Steps use checkbox syntax.

**Goal:** Runnable diagnostic CLI that bootstraps small indicator protocols k times, computes mean/variance per gate, flags high-variance or low-pass as disorders, and emits an outcome-gate matrix with per-trajectory explanations.

**Architecture:** LiteLLM client → raw_tool_loop scaffold → indicator packs (hello_metacog, overeager_mini, slop_indicator) → per-trial metric scorers with explanations → bootstrap aggregator (mean/var/disorder) → markdown report + JSON.

**Tech Stack:** Python 3.11+, pydantic v2, litellm (optional at runtime), pytest, click/typer.

## Global Constraints

- Indicator protocols only — NO full SlopCodeBench / OverEager-Bench suites
- Every metric must be computable from a single TrialTrace with human-readable explanation
- Bootstrap k trials (default 5): report mean, std, and disorder flag when std > threshold OR mean fail rate high
- Outcome gates matrix: dimension × {pass_rate, mean, std, status}
- Scaffold card always recorded (model × scaffold × permission)
- Offline/mock mode for tests without API keys
- Package under `src/dsm_ae/`, tests under `tests/`

## Design: bootstrap disorder rule

```
status = PASS if pass_rate >= 0.8 and std <= 0.25
status = UNSTABLE (disorder) if std > 0.25  # high variance
status = FAIL (disorder) if pass_rate < 0.8
# UNSTABLE and FAIL both count as disorder present
```

Binary trial outcomes use 0/1; continuous metrics normalize or use raw mean/std with separate thresholds.
